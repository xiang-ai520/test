# Phase 3 运行与验证指南

## 本阶段产物

- `learn/app/ai-gateway/`：Node.js WebSocket AI 网关（mock 流式，无真实模型）
- `learn/app/frontend/`：AI 会话页路由 **`/ai`**（文本、摄像头预览、截图发送）

协议说明见 **`docs/phase-3.md`**。

## 先决条件

- Node.js、npm
- 本机已允许浏览器使用摄像头（截图与预览依赖视频轨）

## 启动步骤

### 1. 启动 AI 网关

在 `learn/app/ai-gateway`：

```bash
npm install
npm run dev
```

默认监听 **`ws://0.0.0.0:8790`**。若提示端口占用，请先结束占用 **8790** 的旧进程后再启动。

### 2. 启动前端

在 `learn/app/frontend`：

```bash
npm install
npm run dev
```

默认 **`http://本机IP:5173`**（已绑定 `0.0.0.0`，便于手机局域网访问）。

### 3. 进入会话页

- 浏览器打开前端首页，点击 **「AI 会话（文本 / 图像 / 语音）」**，或直接访问 **`/ai`**。
- 点击 **「连接 AI 网关」**，确认连接状态为已连接。

## 验证步骤

### 验证 1：连接网关

- 能成功连接，无持续报错。
- 若网关未启动，页面应提示无法连接（符合预期）。

### 验证 2：文本消息

- 输入短句并发送。
- 对话区出现用户消息；助手侧为 **分条流式** 字块拼接成的 mock 回复，最后结束为一轮完整气泡。

### 验证 3：截图帧

- 摄像头预览正常时，点击 **「截图并发送」**。
- 对话区有发送记录；网关控制台日志中有 `input.image` 与会话、mime、长度等信息。

### 验证 4（可选）：自动截帧

- 勾选 **「每 5 秒自动截帧并发送」**，观察多次发送与网关日志。

### 验证 5：错误路径

- 网关运行中，可用工具发送非法 JSON 或缺少 `sessionId` 的消息，应收到 **`response.error`**，且网关进程不崩溃（开发自测即可）。

## Phase 3 完成标准

以下全部满足可视为 Phase 3 通过：

1. 前端能稳定连接 AI 网关 WebSocket。
2. 文本能发送，且 mock 流式回复在前端正确拼接。
3. 截图能发送，网关能识别并打日志。
4. 明显错误输入能返回 `response.error`。

## 与 Phase 4 的关系

同一 **`/ai`** 页面在 Phase 4 中增加了语音录音与播放能力；Phase 3 验收可不开启语音功能。完整语音协议见 **`docs/phase-4.md`** 与 **`docs/phase-4-verification.md`**。
