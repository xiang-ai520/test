# Phase 1 运行与验证指南

## 本阶段产物
当前已在 `learn/app` 下生成：

- `frontend/`：React + Vite H5 前端
- `signaling/`：Node.js WebSocket 信令服务

## 目录

```text
learn/app/
  frontend/
  signaling/
```

## 先决条件
你后续准备验证 Phase 1 时，需要本机具备：
- Node.js
- npm

当前我还没有替你安装任何环境。

## 局域网权限风险说明
- `http://电脑局域网IP:5173` 下，摄像头/麦克风权限在不同手机浏览器上的表现可能不同。
- Android Chrome 往往更容易成功。
- iPhone Safari 更可能要求 HTTPS 安全上下文。
- 因此本阶段建议先做：本机双标签页验证，再做 Android 真机验证；如果手机权限失败，下一步再补局域网 HTTPS 方案。

## 启动步骤

### 1. 启动信令服务
在 `learn/app/signaling` 下安装依赖并启动：

```bash
npm install
npm run dev
```

默认监听：
- `ws://0.0.0.0:8788`

### 2. 启动前端
在 `learn/app/frontend` 下安装依赖并启动：

```bash
npm install
npm run dev
```

默认访问：
- 本机：`http://localhost:5173`
- 局域网：`http://你的电脑局域网IP:5173`

## 验证顺序

### 第一步：本机页面可打开
检查：
- 是否能打开加入页
- 是否能输入房间号并进入房间页
- 是否会弹出摄像头/麦克风权限请求

### 第二步：本机双标签页验证
检查：
- 两个标签页输入同一个房间号
- 双方都能看到彼此视频
- 双方都能听到彼此音频

### 第三步：手机局域网验证
检查：
- 手机和电脑连接同一 WiFi
- 手机浏览器访问 `http://电脑局域网IP:5173`
- 手机进入同一个房间
- 手机与电脑之间可以互看/互听

### 第四步：断线验证
检查：
- 一方关闭标签页
- 另一方应看到远端离开状态
- 重新进入后可再次建立连接

### 第五步：控制栏验证
检查：
- 麦克风开关可用
- 摄像头开关可用
- 切换前后摄可用（手机上重点验证）
- 离开房间后返回首页

## Phase 1 通过标准
以下项全部通过，才允许进入 Phase 2：

1. 页面可正常访问。
2. 摄像头和麦克风权限可正常申请。
3. 同房间两端能完成视频通话。
4. 音频与视频都正常。
5. 断开后另一端能感知。
6. 重新加入后能重新连接。
7. 手机真机在局域网下可用。

## 如果验证失败，优先排查
1. Node.js 是否安装。
2. 前端和信令服务是否都成功启动。
3. 手机和电脑是否处于同一 WiFi。
4. Windows 防火墙是否拦截端口。
5. 浏览器是否拒绝了摄像头/麦克风权限。
