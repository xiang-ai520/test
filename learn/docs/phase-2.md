# Phase 2：局域网直播能力

## 阶段目标
在 Phase 1 已稳定的前提下，为系统增加直播能力：
- 主播开播
- 观众观看
- 主播与观众角色分离
- 局域网内稳定播放

## 推荐技术路线
建议使用本地部署的媒体服务：**SRS**。

原因：
- 本地部署简单
- 支持直播常见协议
- 后续可扩展低延迟方案
- 更适合从局域网逐步扩展到外网

## 本阶段范围

### 前端
- 主播端直播控制页
- 观众端直播观看页
- 直播状态提示
- 开播 / 停播操作

### 后端与媒体层
- 媒体服务接入
- 直播房间信息管理
- 主播状态广播
- 观众进入/退出统计（可先做简版）

## 本阶段不做
- 连麦互动
- 观众上麦
- 礼物系统
- 弹幕系统
- 录制回放

## 建议角色模型
- `host`：主播
- `audience`：观众

## 页面建议

### HostLivePage
- 开始直播
- 停止直播
- 本地预览
- 当前状态显示

### AudienceLivePage
- 输入直播房间
- 播放直播流
- 显示主播在线/离线状态

## 架构流向

```text
主播 H5
  └─ 采集摄像头/麦克风
      └─ 推到本地媒体服务

观众 H5
  └─ 从本地媒体服务拉流播放
```

## 与 Phase 1 的关系
- Phase 1 解决 1v1 通话
- Phase 2 解决一对多观看

两者虽然都涉及音视频，但媒体路径不同：
- 通话更偏实时互动
- 直播更偏稳定分发

## 验证清单
以下项全部通过，Phase 2 才算完成：

1. 主播可以开始直播。
2. 主播停止直播后，状态能正确同步。
3. 局域网内另一台设备可以观看直播。
4. 观众端播放稳定，不出现持续黑屏。
5. 直播结束后，观众能看到结束提示。

## 常见风险
- 编解码兼容性
- 手机浏览器自动播放限制
- 播放器协议选择带来的兼容差异
- 媒体服务端口未开放

## 通过门禁
必须确认：
- 主播开播/停播可用
- 观众稳定观看可用
- 真机局域网访问可用

通过后再进入 Phase 3。
