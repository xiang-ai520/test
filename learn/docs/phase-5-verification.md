# Phase 5 运行与验证指南

## 先决条件

- 已按 Phase 3 启动 **`ai-gateway`**（**8790**）与前端。
- 使用 **`/ai`** 页面，并成功 **连接 AI 网关**。
- **重启过 `ai-gateway`**，确保加载含 `SessionStore.frameEvents` 与 `getVisionContextLine` 的版本。

## 验证 1：定时截帧

1. 勾选 **「定时自动截帧」**，间隔选 **3 秒**。
2. 等待至少 2 个周期。
3. 对话区应出现带 **「图像·定时」** 的用户气泡（或等价文案）。
4. 网关日志中 `input.image` 应带 **`timer`** 来源。

## 验证 2：手动与事件截帧

1. 点击 **「手动截图发送」**，再点 **「事件截帧」**。
2. 用户气泡应区分 **手动 / 事件**（与前端展示一致）。
3. 助手 mock 回复中宜出现累计截帧或上下文类描述（流式拼接）。

## 验证 3：文本联合画面上下文

1. 至少发送 **1 帧** 图像后，点击 **「请求一句话摘要（mock）」**。
2. 助手流式正文中应出现 **「【画面上下文】…」** 类句子（无截帧历史时不应出现）。

## 验证 4：语音联合画面上下文

1. 先发 **1～2 帧** 图像。
2. 按 Phase 4 流程 **录音并上传**。
3. 在 **STT 转写之后** 的 mock 流式文本中，同样应出现 **画面上下文** 说明。

## 完成标准

- 定时 / 手动 / 事件三种来源均能上传并在网关记录。
- 文本与语音后的 mock 回复均能 **引用近期截帧上下文**（仍为占位逻辑，非真实识图）。

## 性能说明

截帧频率越高、单帧越大，JSON + base64 负载越大。验证时建议间隔 ≥3s，与开发机性能匹配即可。
