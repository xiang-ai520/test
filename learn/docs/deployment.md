# 局域网部署与后续外网扩展说明

## 当前阶段目标
当前先服务于局域网开发与验证：
- 手机和电脑连接同一 WiFi
- 手机通过电脑局域网 IP 访问 H5 页面
- 信令服务与后续 AI 网关也通过局域网 IP 暴露

## 局域网访问方式
后续运行服务后，典型访问方式会类似：

- 前端页面：`http://电脑局域网IP:前端端口`
- 信令服务：`ws://电脑局域网IP:信令端口`
- AI 网关：`ws://电脑局域网IP:AI网关端口`
- 直播服务：`http://电脑局域网IP:直播端口`

## 局域网验证前提
1. 手机和电脑在同一 WiFi
2. 电脑防火墙允许对应端口访问
3. 开发服务器监听的是 `0.0.0.0` 或局域网地址，而不是只监听 localhost

## 开发期建议
为了减少复杂度，建议 Phase 1 先按以下顺序验证：

1. 电脑本机浏览器访问前端页面
2. 电脑本机双标签页验证通话流程
3. 手机通过局域网地址访问前端页面
4. 手机加入通话房间

## 未来外网扩展
当前不做外网，但架构会预留以下扩展点：

### 1. TURN / STUN
外网视频通话通常需要：
- STUN：协助发现公网地址
- TURN：在直连失败时中继媒体流

### 2. HTTPS
移动端媒体权限在外网环境下通常更依赖 HTTPS。

### 3. 域名与反向代理
后期可能需要：
- 域名
- Nginx / Caddy 反向代理
- WSS / HTTPS 统一入口

### 4. 服务编排
后期建议用 Docker Compose 或类似方式统一管理：
- frontend
- signaling
- media
- ai-gateway
- turn

## 当前局域网优先的好处
- 问题更少
- 延迟更低
- 便于快速验证功能正确性
- 避免一开始就陷入 NAT 穿透与证书问题

## 阶段建议
- Phase 1 到 Phase 5：以局域网可用为核心目标
- Phase 6：再正式做外网接入与生产化增强
